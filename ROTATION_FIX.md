# Исправление поворота игрока

## Дата: 26 ноября 2025

## Проблема

После первой реализации игрок стоял спиной к курсору вместо того чтобы смотреть на него.

## Причина

Использовалась неправильная компенсация угла:
- **Было:** `angleToMouse + 90.0f` (предполагалось что текстура смотрит вниз)
- **На самом деле:** Текстура смотрит вверх

## Решение

Изменена компенсация угла с +90° на -90°:

### Сервер (Zero_Ground/Zero_Ground.cpp)
```cpp
// Было:
serverSprite.setRotation(angleToMouse + 90.0f);

// Стало:
serverSprite.setRotation(angleToMouse - 90.0f);
```

### Клиент (Zero_Ground_client/Zero_Ground_client.cpp)
```cpp
// Было:
clientSprite.setRotation(angleToMouse + 90.0f);

// Стало:
clientSprite.setRotation(angleToMouse - 90.0f);
```

## Объяснение

### Система координат SFML:
- **0°** = направление вправо (→)
- **90°** = направление вниз (↓)
- **180°** = направление влево (←)
- **-90°** (или 270°) = направление вверх (↑)

### Ориентация текстуры:
- Текстура `Nothing_1.png` изначально смотрит **вверх** (↑)
- Это соответствует углу **-90°** в системе SFML
- Поэтому нужно **вычитать** 90° из вычисленного угла

### Примеры:
| Направление мыши | Угол atan2 | Компенсация | Итоговый угол | Результат |
|------------------|------------|-------------|---------------|-----------|
| Вправо (→)       | 0°         | -90°        | -90°          | Вправо ✓  |
| Вниз (↓)         | 90°        | -90°        | 0°            | Вниз ✓    |
| Влево (←)        | 180°       | -90°        | 90°           | Влево ✓   |
| Вверх (↑)        | -90°       | -90°        | -180°         | Вверх ✓   |
